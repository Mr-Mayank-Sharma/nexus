## Prerequisites

- Docker 24+ and Docker Compose v2

Copy `.env.example` to `.env` and configure:

| Variable | Required | Default | Description |
|---|---|---|---|
| `SPRING_DATASOURCE_URL` | Yes | `jdbc:postgresql://localhost:5432/nexus` | PostgreSQL JDBC URL |
| `SPRING_DATASOURCE_USERNAME` | Yes | `nexus` | Database user |
| `SPRING_DATASOURCE_PASSWORD` | Yes | â€” | Database password |
| `JWT_SECRET` | Yes | â€” | 256-bit JWT signing key (min 32 chars) |
| `JWT_EXPIRATION_MS` | No | `86400000` | Token lifetime (ms) |
| `REDIS_HOST` | No | `localhost` | Redis host |
| `REDIS_PORT` | No | `6379` | Redis port |
| `KAFKA_BOOTSTRAP_SERVERS` | No | `localhost:9092` | Kafka broker list |
| `SPRING_PROFILES_ACTIVE` | Yes | `prod` | Spring profile |
| `CORS_ALLOWED_ORIGINS` | Yes | `https://your-domain.com` | Comma-separated allowed origins |

### Backend Security Headers

The following headers are set automatically by `SecurityHeadersFilter`:

| Header | Value |
|---|---|
| `Strict-Transport-Security` | `max-age=31536000; includeSubDomains; preload` |
| `X-Content-Type-Options` | `nosniff` |
| `X-Frame-Options` | `DENY` |
| `X-XSS-Protection` | `0` (modern browsers use CSP instead) |
| `Referrer-Policy` | `strict-origin-when-cross-origin` |
| `Permissions-Policy` | `camera=(), microphone=(), geolocation=()` |
| `Content-Security-Policy` | Configured via `SecurityConfig` |

## 2. Build

### Backend



## 6. Frontend PWA

The frontend is a Progressive Web App:

- **Service Worker**: Auto-generated by `vite-plugin-pwa` (Workbox)
- **Offline**: OfflinePage shown when network is unavailable
- **Update**: UpdateBanner appears when new SW is detected
- **Icons**: 8 sizes (72pxâ€“512px) in `public/icons/`
- **Manifest**: Inline in `vite.config.ts`

### Precaching

162 entries (~2.3MB) precached by service worker on first load.

### Runtime Caching

| Pattern | Strategy | TTL |
|---|---|---|
| Google Fonts CSS | CacheFirst | 1 year |
| Google Fonts WOFF2 | CacheFirst | 1 year |
| `/api/*` | NetworkFirst | 1 day (10s timeout) |

## 7. Security Checklist

- [ ] `JWT_SECRET` is a strong random string (â‰Ą32 chars)

- [ ] HTTPS enforced at load balancer / reverse proxy
- [ ] HSTS header enabled (automatic via SecurityHeadersFilter)
- [ ] CSP configured for your domain
- [ ] Default admin password changed
- [ ] Database not exposed to public network
- [ ] Redis password set and not exposed
- [ ] Kafka not exposed to public network
- [ ] `.env` file not committed to git

## 8. Nginx Configuration

```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/ssl/certs/nexus.pem;
    ssl_certificate_key /etc/ssl/private/nexus.key;

    # Frontend SPA
    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # API proxy
    location /api/ {
        proxy_pass http://nexus-backend:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Cache static assets
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # Security headers
    add_header X-Frame-Options "DENY" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
}
```

## 9. Scaling

### Horizontal Scaling

- Backend: Stateless (JWT auth), scale via Kubernetes replicas or Docker Compose `--scale`
- Frontend: Static files, serve from CDN (CloudFront, Cloudflare)
- PostgreSQL: Read replicas for analytics queries
- Redis: Cluster mode for >10k concurrent sessions

## 10. Troubleshooting

### Backend won't start

```bash
# Check Java version
java -version  # Should be 17.x

# Check database connectivity
psql -h localhost -U nexus -d nexus -c "SELECT 1"

# Check Flyway migrations
grep " flyway" application.yml
```

### Frontend build fails

```bash
# Clear cache
rm -rf node_modules dist
npm ci
npx vite build
```

### Service Worker issues

```bash
# Clear SW cache in browser
# Chrome: Application > Service Workers > Unregister
# Or: hard refresh (Ctrl+Shift+R)
```
